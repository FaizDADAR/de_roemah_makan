## 🚀 Kontribusi
Untuk menjaga kualitas kode, setiap perubahan wajib mengikuti standar:
1. Menggunakan **Conventional Commits** (e.g., `feat:`, `fix:`, `chore:`).
2. Memperbarui dokumentasi terkait jika ada perubahan skema database atau alur bisnis.
3. Selalu melakukan pengujian di lingkungan **Docker** sebelum di-push ke main branch.
